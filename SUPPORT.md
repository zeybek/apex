# Support

Thanks for using Apex. Here is where to go for help.

## Questions and usage help

Check the [README](README.md), [adapters/README.md](adapters/README.md), and [evals/README.md](evals/README.md) first — they cover installation, per-client setup, and evaluation. If that does not answer it, open a [GitHub issue](https://github.com/zeybek/apex/issues).

## Bugs and feature requests

Use the issue forms:

- a skill that mis-triggers, a validator false positive, or a packaging problem → **Bug report**;
- a new skill, a reference, or a workflow improvement → **Feature request**.

## Security issues

Do not open a public issue. Follow [SECURITY.md](SECURITY.md) and email **me@zeybek.dev**.

## What to expect

This is a single-maintainer project. Issues are triaged on a best-effort basis; security reports are prioritized on the timeline in [SECURITY.md](SECURITY.md). There is no guaranteed response time for non-security issues.
