# Service Boundary Decision

Use `apex-design` to decide whether billing should leave a growing modular monolith.

## Learning Goal

Produce an implementable decision that turns broad reliability and audit concerns into measurable scenarios, compares credible alternatives including retaining the current design, and defines a reversible delivery path.

## Run

1. Copy [`workspace/`](workspace/) to a temporary directory.
2. Start a clean agent session in that directory with `apex-design`.
3. Provide [`prompt.md`](prompt.md) as the task.
4. Evaluate the result with [`rubric.md`](rubric.md).

This walkthrough is judgment-based and has no mechanical verifier.
